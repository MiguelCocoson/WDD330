Week 3 Notes

Chapter 5: Objects

  • Object: self-contained set of related values and functions.
  • Creates an empty object: new Object();
  • Objects work similarly to dictionaries in python:
  • "object.key" gives the value.
  • "object[value]" also returns the value.
  • "undifined" is the error obtain when trying to acces a property that does not exist.

Chapter 6: Document Object Model

  • DOM: Document Object Model
  • DOMs represents an HTML document as a network of connected nodes that form a tree-like structure.
  • We can use this to access parts of the HTML in the javascript file:
  • "Document.body" returns the body element of a web page, as we saw in the previous example.
  • "Document.images" returns a node list of all the images contained in the document.
  • "Document.links" returns a node list of all the <a> elements and <area> elements that have an href attribute.
  • "Document.anchors" returns a node list of all the <a> elements that have a name attribute.
  • "Document.forms" returns a node list of all the forms in the document. This will be used when we cover forms in Chapter 8.

Chapter 7: Events

  • Events are the connection between the website and the user input or interactions.
  • "click" The click event occurs when a user clicks with the mouse, presses the Enter key, or taps the screen.
  • "type" The type property returns the type of event that occurred.
  • "target" The target property returns a reference to the node that fired the event.
  • "dblclick" double click.
  • "keydown" Press and hold.
  • "keypress" When key is press.
  • "keyup" When key is release.
